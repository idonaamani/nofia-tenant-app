# הקמת Supabase — מדריך צעד-צעד

מדריך זה מיועד לרגע שבו תרצו להפעיל את מערכת ההתחברות המאובטחת (וילה+ת.ז בכניסה ראשונה, סיסמה בכניסות הבאות, ופאנל ניהול). עד אז האפליקציה ממשיכה לעבוד בדיוק כמו היום, עם הכניסה הפשוטה הישנה.

**עלות: ₪0** — כל השירותים כאן בתוכנית החינמית, בלי כרטיס אשראי.

## שלב 1 — יצירת פרויקט Supabase

1. היכנסו ל-https://supabase.com ולחצו "Start your project".
2. הירשמו עם מייל או GitHub.
3. צרו פרויקט חדש (New Project) — תנו לו שם כמו `nofia-tenant-app`, בחרו סיסמת מסד נתונים (שמרו אותה במקום בטוח, לא נצטרך אותה שוב בשימוש שוטף), ובחרו אזור קרוב (למשל Frankfurt/eu-central).
4. המתינו כ-2 דקות עד שהפרויקט מוכן.

## שלב 2 — הרצת סכימת מסד הנתונים

1. בתפריט הצד: **SQL Editor** → **New query**.
2. פתחו את הקובץ `supabase/migrations/0001_init.sql` מהפרויקט, העתיקו את כל התוכן, הדביקו בעורך, ולחצו **Run**.

## שלב 3 — קבלת מפתחות החיבור

1. בתפריט הצד: **Project Settings** → **API**.
2. העתיקו את **Project URL** ואת **anon public key**.
3. בתיקיית הפרויקט, צרו קובץ בשם `.env.local` (העתק של `.env.example`) ומלאו:
   ```
   VITE_SUPABASE_URL=<ה-Project URL שהעתקתם>
   VITE_SUPABASE_ANON_KEY=<ה-anon key שהעתקתם>
   ```
4. הריצו מחדש את השרת המקומי (`npm run dev`) — מסך הכניסה אמור להשתנות אוטומטית למסך המאובטח.

## שלב 4 — פריסת ה-Edge Functions

צריך להתקין את ה-CLI של Supabase פעם אחת (יש לו הוראות התקנה למחשב שלכם ב-https://supabase.com/docs/guides/cli), ואז מתיקיית הפרויקט:

```
supabase login
supabase link --project-ref <project-ref-מהכתובת-של-הפרויקט>
supabase functions deploy tenant-verify-id
supabase functions deploy tenant-set-password
supabase functions deploy admin-reset-password
supabase functions deploy admin-delete-villa
```

לפני הפריסה, יש להגדיר את מפתח ה-service role כ"סוד" (secret) עבור הפונקציות — הוא **לא** מוזן ב-`.env.local` (זה מפתח רגיש שלא אמור להגיע לדפדפן בשום צורה):

1. Project Settings → API → העתיקו את **service_role key**.
2. הריצו: `supabase secrets set SUPABASE_SERVICE_ROLE_KEY=<המפתח שהעתקתם>`

## שלב 5 — יצירת המנהלים הראשונים (אתה/לוטוס/מורן/דולב)

1. הריצו את האפליקציה, גשו לכתובת עם `?admin` בסוף (למשל `http://localhost:5173/?admin`).
2. יופיע מסך "אין הרשאת מנהל" בניסיון הראשון — זה תקין, כי עדיין לא נרשמתם. תחת Authentication → Users בפרויקט ה-Supabase, לחצו **Add user** עבור כל אחד מארבעתכם (מייל אמיתי + סיסמה).
3. עבור כל מנהל, העתיקו את ה-`User UID` שלו מרשימת המשתמשים, וב-SQL Editor הריצו (פעם אחת לכל מנהל):
   ```sql
   insert into public.admins (user_id, name) values ('<ה-UID שהעתקתם>', 'ידו');
   ```
4. כעת ניתן להתחבר במסך `?admin` עם אותו מייל+סיסמה.

## שלב 6 — הוספת הוילות והדיירים

הכי מהיר: ב-SQL Editor הריצו את כל התוכן של `supabase/seed_villas.sql` — הוא כבר מכיל את 24 הוילות עם השמות, הת.ז (של שני בני הזוג איפה שרלוונטי), טלפון ומייל, ישירות מתוך "דף קשר בעלי מגרשים.xlsx". (חלופה: להוסיף ידנית דרך פאנל הניהול `?admin` → "+ הוספת וילה" לכל וילה בנפרד.)

הדיירים עדיין לא יכולים להתחבר בשלב הזה — כל אחד מבני הזוג צריך לעבור בעצמו את "כניסה ראשונה" (וילה + ת.ז שלו) כדי לבחור סיסמה אישית.

## שלב 7 — פרסום האפליקציה באינטרנט (כדי שדיירים יוכלו להיכנס מהבית)

היום האפליקציה רצה רק על `localhost` במחשב שלך. כדי שדיירים יוכלו להיכנס מהטלפון/מחשב שלהם:

1. היכנסו ל-https://vercel.com, הירשמו (חינם, בלי כרטיס אשראי).
2. חברו את תיקיית הפרויקט (או ה-repo אם יש לכם ב-GitHub) ולחצו Deploy.
3. **חשוב:** בהגדרות הפרויקט ב-Vercel → Environment Variables, הוסיפו את אותם שני משתנים מ-`.env.local` (`VITE_SUPABASE_URL`, `VITE_SUPABASE_ANON_KEY`) — אחרת האתר המפורסם יעבוד במצב הישן הלא-מאובטח.

## רשימת בדיקה לפני מסירה לדיירים

- [ ] נכנסתי כמנהל ב-`?admin` והוספתי וילה לדוגמה.
- [ ] מהמסך הראשי (לא `?admin`), עשיתי "כניסה ראשונה" עם מספר הוילה + הת.ז שהזנתי, בחרתי סיסמה, ונכנסתי בהצלחה לאשף.
- [ ] יצאתי ונכנסתי שוב עם מספר וילה + הסיסמה שבחרתי — עובד.
- [ ] בפאנל הניהול לחצתי "איפוס סיסמה" לאותה וילה, ווידאתי שכניסה עם הסיסמה הישנה כבר לא עובדת, וצריך לעבור שוב "כניסה ראשונה".
- [ ] בדקתי שהזנת ת.ז שגויה כמה פעמים ברציפות חוסמת זמנית (429 / הודעת "יותר מדי ניסיונות").
