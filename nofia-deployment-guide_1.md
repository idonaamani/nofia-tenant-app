# מדריך: איך הופכים את "נופיה אלפי מנשה" לאתר/אפליקציה חי בחינם

יש לך כרגע שני חלקים נפרדים:
1. **אפליקציית שינויי הדיירים** – קובץ HTML/JSX סטטי (רץ בדפדפן, לא צריך שרת).
2. **בוט הטלגרם** – שרת Node.js שצריך לרוץ כל הזמן ("שרת חי") כדי לקבל בקשות.

לכל אחד מסלול פריסה (deployment) קצת שונה. הנה השלבים בסדר מומלץ.

---

## שלב 1: להעלות את הקוד ל-GitHub (חינם)

זה הבסיס לכל השאר – כל שירותי האירוח החינמיים מתחברים ל-GitHub.

1. פתח/י חשבון בחינם ב-[github.com](https://github.com) אם עדיין אין לך.
2. צור/י repository חדש (למשל בשם `nofia-tenant-app`).
3. העלה/י לתוכו את כל הקבצים מתיקיית `nofia-tenant-app` שלך (כולל התיקייה `telegram-bot` אם תרצה גם אותה שם, או ב-repo נפרד).
   - הכי קל: גרור/י את הקבצים ישירות בממשק הווב של GitHub ("Add file" → "Upload files"), בלי צורך בפקודות טרמינל.
   - **חשוב:** אל תעלה את קובץ `.env` האמיתי (עם הטוקן) — רק את `.env.example`.

---

## שלב 2: פריסת אפליקציית שינויי הדיירים (חינמי לגמרי, קל)

זה הקובץ הסטטי (HTML) – אין לו תלות בשרת, אז זה הכי פשוט.

**אפשרות מומלצת: Netlify**
1. הרשמ/י בחינם ב-[netlify.com](https://netlify.com) (אפשר להתחבר ישירות עם GitHub).
2. "Add new site" → "Import an existing project" → בחר/י את ה-repo מ-GitHub.
3. Netlify יזהה שזה אתר סטטי ויפרוס אותו אוטומטית.
4. תוך דקה תקבל/י כתובת חינמית כמו `nofia-tenant.netlify.app`.
5. כל פעם שתעדכן/י את הקוד ב-GitHub, האתר יתעדכן אוטומטית.

**אפשרות חלופית:** Vercel או GitHub Pages – עובדות כמעט אותו דבר.

**דומיין אישי (אופציונלי, לא חובה):** אפשר לקנות דומיין (למשל `nofia-alfei-menashe.co.il`, בד"כ 30-70 ₪ לשנה) ולחבר אותו ל-Netlify בהגדרות ("Domain settings") — האתר עצמו נשאר חינמי, רק הדומיין עולה כסף.

---

## שלב 3: פריסת בוט הטלגרם (שרת חי, צריך תשומת לב)

בוט טלגרם צריך שרת שרץ **24/7**, לא רק קובץ סטטי. יש שירותי ענן עם רמה חינמית:

**אפשרות מומלצת: Render.com**
1. הרשמ/י בחינם ב-[render.com](https://render.com) עם GitHub.
2. "New" → "Web Service" → בחר/י את ה-repo (או תיקיית `telegram-bot` בתוכו).
3. הגדרות בנייה: `npm install` (Build Command), `npm start` (Start Command).
4. בממשק Render, תחת "Environment" – הוסף/י את משתני הסביבה **ידנית** (לא מהקובץ .env!):
   - `BOT_TOKEN`
   - `MANAGEMENT_CHAT_ID`
   - `APP_SHARED_SECRET`
5. תקבל/י כתובת ציבורית כמו `https://nofia-bot.onrender.com`.
6. עדכן/י את `BOT_SERVER_URL` בקובץ `telegram-integration-snippet.js` (ובתוך האפליקציה עצמה) לכתובת הזאת במקום `localhost`.

⚠️ **הערה חשובה על התוכנית החינמית של Render:** שרת בתוכנית החינמית "נרדם" אחרי כ-15 דקות בלי תנועה, וה"התעוררות" הראשונה לוקחת כ-30-60 שניות. זה אומר שהודעת האישור הראשונה עלולה להתעכב קצת. אם זה קריטי, אפשר בהמשך לשדרג לתוכנית בתשלום קטנה (כ-$7/חודש) שלא נרדמת.

## שלב 4: לחבר בין שני החלקים

לאחר ששני החלקים חיים באינטרנט:
1. פתח/י את קובץ האפליקציה (או קוד המקור אצל אנטי גרוויטי).
2. ודא/י שכתובת ה-`fetch` בכפתור "שלח לאישור מנהל פרויקט" מצביעה לכתובת ה-Render האמיתית, לא ל-`localhost`.
3. העלה/י את הקובץ המעודכן חזרה ל-GitHub → Netlify יתעדכן אוטומטית.
4. בדוק/י מקצה לקצה: מלא/י בחירות באתר החי → לחצ/י "שלח לאישור" → ודא/י שההודעה מגיעה לטלגרם.
